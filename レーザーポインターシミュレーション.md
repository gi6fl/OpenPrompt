# レーザーポインターシミュレーション

以下の指示に従い、「レーザーポインターシミュレーション」を作成してください。

## 技術スタック

- **Vite** + **TypeScript**（エージェントが即座にビルド・バグ修正できること）
- **Three.js** 最新安定版

## 重要な設計指針

- 形状は**3D立体**として描画する。床面や壁のような2D平面にレーザーが反射してはならない。
- レーザーは立体の内側の面にも反射する。
- 形状は半透明（opacity: 0.4程度、DoubleSide）で描画するが、レイキャストには半透明設定が影響しないことを確認する。
- レイキャストは毎フレーム実行し、古いワールド座標での交差判定を防ぐ。

## 形状

- 立方体: 6面の3D立体。
- 三角柱: 5面の3D立体。
- 球体: 球体の3D立体。
- 立体はすべて半透明メッシュ + 縁取りワイヤーフレーム。
- 複数追加可能。位置(X/Y/Z)、スケール、回転(X/Y/Z)をスライダーでリアルタイム変更。
- スライダーのレンジはかなり広く。

## 物理計算

- Three.jsのRaycasterでレイとメッシュの交差判定。
- 反射ベクトルはThree.js標準機能で計算。
- 自己交差防止のため、反射後の起点を交差点から微小オフセットする。

## レーザー

- 発射源は簡易的なレーザーポインター風3Dオブジェクト。
- 位置(X/Y/Z)、発射方向(X/Y/Z)をスライダーで変更。
- ポインターのジオメトリは +Z 軸方向に配置すること。

## UI

- ダークテーマ (背景: #0a0a0a、アクセント: #00ffcc)
- フォントは **Noto Sans** を使う
- レイアウト: 左に3Dキャンバス、右にサイドパネル
- サイドパネル: オブジェクト追加ボタン(立方体/三角柱)、リセットボタン、レーザー設定、オブジェクトリスト、選択オブジェクトのプロパティ
- OrbitControlsで3Dビューの回転・ズーム
- WASDでカメラ**移動のみ**可能
- UIのすべてのテキストは**日本語**にする


## 品質要件

- Three.jsの初期化は1回だけ実行する。
- UIスライダーの変更は毎フレームのレンダリングに即時反映される。
- リサイズ対応。

## 完了判定

- `npm run build` でビルドエラーなし。
- コードに明らかなランタイムエラーがないことを確認する。
