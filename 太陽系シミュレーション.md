# 太陽系シミュレーション

ブラウザで動作する、物理的に正確でビジュアル的に美しい太陽系のリアルタイム 3D シミュレーションを実装してください。

**最重要方針**: これは「それっぽい演出」ではなく**実際の太陽系の N 体物理シミュレーション**です。距離・質量・軌道要素・公転周期は実測値をそのまま使い、デフォルメは描画レイヤーのみに閉じ込めます。地球がちょうど 1 年で太陽を 1 周すれば、物理は正しく実装できています。

---

## 1. 技術スタック

- **Vite + TypeScript**（strict、`verbatimModuleSyntax` / `noUnusedLocals` / `noUnusedParameters` 有効）
- **Three.js**（最新版）+ `@types/three`（devDependency として必須。three 本体に型は同梱されない）
- 外部アセット禁止。テクスチャはすべてプロシージャル生成（Canvas 2D）

---

## 2. 物理シミュレーション

### 単位系

距離 = **AU**、質量 = **太陽質量 M☉**、時間 = **年**。この単位系では重力定数は **G = 4π²** になり、検算が容易（地球: a=1 AU, 周期=1 年）。

### N 体計算

- 太陽 + 水星〜海王星 + 冥王星の **10 天体**、全ペア総当たりの重力計算。
- 天体 a が b から受ける加速度は `G · m_b · r̂ / r²` として**天体ごとに独立に**計算する。作用・反作用を 1 つの式にまとめて質量差で書くような「最適化」は、太陽質量 ≈ 1 に対し惑星質量が極小なため引力が桁違いに弱くなり軌道が破綻する。絶対にやらない。
- **太陽も自由運動させる**（原点固定しない）。初期化時に惑星の全運動量を打ち消す初期速度を太陽に与え、系の重心が漂わないようにする。
- 近接時の特異点回避として微小な softening を入れてよい。

### 数値積分

- **Velocity Verlet 法**を使用する:

```
x(t+Δt) = x(t) + v(t)Δt + ½a(t)Δt²
v(t+Δt) = v(t) + ½(a(t) + a(t+Δt))Δt
```

- dt は積分安定性のため常に小さく固定する。シミュレーション速度（倍速）の調整は dt を大きくするのではなく**サブステップ数**で行う。
- **固定タイムステップ + フレームレート補正**: 毎フレーム、前フレームからの実経過時間に応じてサブステップ数を決める。これでリフレッシュレートに依存せず同じ物理時間で進行する。サブステップ数には上限を設けるが、倍速設定に対して低すぎる上限（数百程度）だと動きが実質停止するため、数千オーダーを確保する。
- デフォルトの時間スケールは「地球の公転が目で追える速さ」（実時間 1 秒 ≈ 数日〜十数日）を目安とする。

### 軌道の初期化（ケプラー軌道要素）

各惑星は **J2000.0 エポックの実測軌道要素**（a, e, i, Ω, ω）から位置・速度を導出する。真近点角 ν は天体ごとに分散させ、初期配置が一直線に並ばないようにする。

```
p = a(1 - e²)
r = p / (1 + e cos ν)
h = √(μp)        （μ = G · M_sun — N体で使う太陽質量と必ず同一の値）

位置 (PQW): (r cos ν, r sin ν, 0)
速度 (PQW): (-(μ/h) sin ν, (μ/h)(e + cos ν), 0)
```

**座標系**: 黄道面 = XZ 平面、Y 軸 = 北黄道極。PQW → 黄道座標の回転は、教科書の Z-up 公式 `R_z(Ω) R_x(i) R_z(ω)` をそのまま使うと軸がずれる。Y-up に合わせて `R_y(Ω) R_x(i) R_z(ω)` 系で導出すること。

**命名**: Ω と ω は取り違えやすい。プロパティ名は `longitudeOfAscendingNode` / `argumentOfPerihelion` のように省略せず書く。

## 3. 描画スケール（物理と見た目の分離）

- 物理状態は AU のまま保持し、**Three.js のシーングラフに反映する時だけ**スケーリングする。軌道距離・速度は一切いじらない。
- 惑星の半径は実比率を保ったまま、全惑星共通の倍率（実半径の数百〜数千倍を目安）を掛けて視認可能にする。数十倍では初期カメラ距離から 1 ピクセル未満になり見えない。逆に大きすぎると距離感が壊れる。
- 太陽だけは実比率で描くと内惑星軌道を覆うため、プロシージャルテクスチャ付きの小さめの球 + **単一スプライト**（放射状グラデーション、白黄→オレンジ→透明、Additive Blending）のグローで表現する。球を何重にも重ねない。

## 4. ビジュアル要件

### プロシージャルテクスチャ（全天体共通）

- 単色や簡易グラデーションは不可。高解像度 Canvas に複数オクターブのノイズ（周波数 2 倍・振幅 ½ ずつ）を重ね、天体ごとの特徴的な地形・帯構造を描く。
- **シームレス化**: UV を 2D ノイズに直接渡すと u=0/1 の境界に継ぎ目が出る。UV → 球面 3D 座標に変換してから **3D ノイズ**でサンプリングすること。
- テクスチャ生成関数は統一インターフェース（必要なマップ一式を返す）で定義する。
- `texture.colorSpace = THREE.SRGBColorSpace` を明示設定する（デフォルトは NoColorSpace で色が薄くなる）。
- CanvasTexture をマテリアルが参照している間に `dispose()` しない。

### 法線マップ

高さマップから生成する。手順を厳守:

1. 勾配 dx は同じ行の `(x-1,y)` と `(x+1,y)` の高さ差、dy は同じ列の `(x,y-1)` と `(x,y+1)` の高さ差（エッジはラップアラウンド）。高さが `[0,255]` なら先に 256 で割って正規化。
2. `[dx·strength, dy·strength, 1.0]` をベクトルとして**正規化**する（省略すると法線が壊れ、反射が破綻する）。
3. `(n * 0.5 + 0.5) * 255` で `[0,255]` に変換。

`eps` で割る数値微分は勾配が数百に爆発するため使わない。法線マップを持たない天体では `normalMap: null` を渡さず、プロパティ自体を省略する（Three.js が警告を出す）。

### 天体ごとの見どころ

| 天体 | 表現すべき特徴 |
|------|---------------|
| 太陽 | プラズマ質感 + グロー |
| 水星 | クレーター地形（法線マップ付き）、暗色平原 |
| 金星 | 黄色い厚い雲の層状・渦パターン、大気リムライト |
| 地球 | 陸海の色分け、独立回転する半透明雲レイヤー、大気リムライト、夜側の都市光 |
| 火星 | 赤い地表、マリネリス渓谷・オリンポス山などの明暗、極冠 |
| 木星 | 縞模様の帯構造、大赤斑 |
| 土星 | 淡い縞 + **環**（カッシーニ分裂を含む透過テクスチャ、両面描画） |
| 天王星 | メタン大気の青緑グラデーション、薄い帯 |
| 海王星 | 青い帯構造、大黒斑 |
| 冥王星 | ハート形のスプートニク平野を明確に |

### 土星の環の実装注意

- `RingGeometry` のデフォルト UV は放射状でない。各頂点の半径から u を計算し直し、テクスチャ横軸 = 環の内→外とする（v は 0.5 固定でよい）。
- 環テクスチャは `LinearSRGBColorSpace` を設定する（SRGB だと透過が正しくならない）。
- **環の回転方向**: `RingGeometry` は XY 平面で生成される。XZ 平面に配置するには `rotation.x = -π/2` だが、この後に軸傾斜を Euler 角で追加すると順序依存の誤差が生じる（`set(x, y, z)` の内部乗算順序が XYZ）。Euler 角の代わりに**クォータニオン**で合成すること:
- 環は対称な円盤なので、惑星の自転角（`rotationAngle`）を環に適用する必要はない。適用すると環が惑星と一緒に回転する誤った見た目になる。

### 大気リムライト

大気を持つ天体（地球・金星）に、`rim = pow(1.0 - max(dot(normal, viewDir), 0.0), k)` のカスタム GLSL シェーダーを BackSide + Additive Blending でかぶせ、縁が発光する大気を表現する。

### 自転・軸傾斜

各天体は実測の自転周期・軸傾斜で回転させる（時間スケールと連動）。地球の雲は地表より速く回す。

### 星空・照明

- 背景に 1 万個超の星パーティクルを球面配置（暖色・寒色・白を混ぜ、Additive Blending）。
- 太陽位置に距離減衰する PointLight。強度は「内惑星が白飛びせず、海王星でも微弱に照らされる」バランス（数十〜百程度 + ACES Filmic トーンマッピングで調整）。環境光は影を完全な黒にしない程度の微弱値。
- 出力カラー空間 sRGB、ACES Filmic トーンマッピング。

## 5. カメラ（6 自由度フリーフライト）

- OrbitControls ではなく自作のフリーフライトカメラ。位置と視線方向（ヨー・ピッチ）が独立。
- **WASD** で視線基準の前後左右移動（ワールド軸に固定しない）、**Q/E**（または Space/Shift）で上下移動。
- **マウスドラッグ**で視点回転。Pointer Lock は使わない。したがって `movementX/Y` は使えない — `mousedown`/`mousemove`/`mouseup` で `clientX/Y` の差分を自前管理する。
- **ホイール**で移動速度またはズームを調整。
- 初期位置は太陽系全体（外惑星軌道まで）が視野に入る俯瞰距離。移動速度は内惑星〜外惑星間を短時間で移動できる、描画倍率と連動しない固定値。

## 6. プロジェクト構造

単一ファイル禁止。役割ごとに分離する:

```
src/
├── main.ts                  # セットアップ + requestAnimationFrame ループのみ（~60行）
├── style.css                # フルスクリーン canvas
├── data/
│   └── bodies.ts            # 天体データ（質量・半径・自転・J2000軌道要素・視覚フラグ）
├── physics/
│   ├── constants.ts         # G, dt, サブステップ, スケール等の物理定数のみ
│   ├── kepler.ts            # 軌道要素 → 状態ベクトル変換
│   └── nbody.ts             # Velocity Verlet N体積分
├── rendering/
│   ├── solarSystem.ts       # 物理初期化 + メッシュ生成 + 毎フレーム同期の統合クラス
│   ├── atmosphere.ts        # 大気リムライトシェーダー
│   ├── starfield.ts         # 星空
│   └── textures/
│       └── planets.ts       # プロシージャルテクスチャ生成（共通ノイズ関数含む）
└── camera/ もしくは controls.ts   # フリーフライトカメラ
```

UI は不要。フルスクリーンの 3D キャンバスのみ、ダークテーマ。Vite テンプレート由来の不要ファイル（`counter.ts` 等）は削除する。`index.html` の `lang` は `"ja"`。

## 7. 完了基準

以下をすべて満たしてから完了報告すること:

1. `npm run build` がエラー・警告ゼロで通る（`import type` の使い分け、未使用宣言なし、イベントハンドラ引数の明示的型注釈を含む）。
2. `npm run dev` で起動し、太陽系全体が俯瞰で描画される。
3. 惑星が実測に整合する速度で公転している（地球 ≈ 1 年/周、水星は速く、海王星は非常に遅い）。長時間放置しても軌道が発散しない。
4. 全惑星のテクスチャがプロシージャルで、継ぎ目がなく、単色でない。
5. 地球の雲・大気グロー・夜景、土星の環（カッシーニ分裂と透過）が正しく表示される。
6. フリーフライトカメラ（WASD + Q/E + ドラッグ回転）が全方向に機能する。
7. プロジェクト構造が上記構成に従い、main.ts がミニマルである。
